# Settings 配置系统实施文档

## 实施概览

本文档基于 [settings.md](../spec/settings.md) 规范，提供完整的实施指导。

### 实施范围

- Zod Schema 定义与类型系统
- 配置加载与深度合并
- 环境变量替换
- 错误处理
- 单元测试

---

## 设计决策回顾

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 配置归属 | 纯服务端 | 所有读写逻辑由 server 处理，前端不感知 |
| API 暴露 | 不暴露 | Settings 是内部实现，无对外 API |
| 加载时机 | 启动时一次性加载 | 简单可靠，变更需重启 |
| 环境变量替换 | 仅字符串值 | 避免类型转换歧义 |
| 配置缺失 | 使用默认值 | 开箱即用，无需强制创建配置文件 |
| 模块位置 | `packages/server/src/config/` | 服务端内部使用，无跨包共享需求 |

## 概要设计

** 核心包的使用方式 **

1. 开发者通过 zod. 预定义 settings 的类型和默认值并初始化实例，然后在入口处执行 实例.load() 用于 加载配置
2. 实例支持 get() 获取配置，支持 set() 修改配置，支持 watch() 监听配置变化
  - get() 支持传参和不传参，不传参时返回所有配置，传参时返回指定配置
  - set() 支持修改指定配置，支持修改所有配置
3. 实例在 set 和 get 时具有类型提示


** 核心包的内部实现 **

1. 配置按照优先级：cli 参数 > 环境变量 > 本地配置 > 用户配置 > 默认值 ，深度合并配置
2. 配置文件位置:
  1. 本地配置：工作目录/.<app_name>/settings.json
  2. 用户配置：~/.config/<app_name>/settings.json、~/.config/<app_name>/credentials.json
  3. 环境变量：环境变量
  4. cli 参数：cli 参数

2. 加载方式：
  1. 加载本地配置并校验
  2. 加载用户配置和用户凭证配置 深度合并后进行校验
  3. 加载环境变量并校验
  4. 加载 cli 参数并校验
  5. 深度合并所有配置
  
3. 校验配置：使用 zod 对配置进行校验，如果校验失败，抛出错误
